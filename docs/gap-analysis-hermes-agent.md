# Gap Analysis: Hermes Agent vs PureClaw

## Missing Functionality Categories

These are major categories of functionality where PureClaw has zero implementation,
not just missing integrations within an existing category. Items like "more channels"
or "more providers" are excluded since PureClaw already has the abstractions.

Items already identified in the OpenClaw gap analysis are noted but not repeated in
detail. New categories unique to Hermes Agent (or significantly different in
approach) are the focus.

### 1. Browser Automation

_(Also missing vs OpenClaw)_

Hermes has `browser_navigate`, `browser_snapshot`, and `browser_vision` tools with
multiple backend options (Browserbase, Browser Use, local Chrome/Chromium via CDP).
PureClaw has no browser control.

### 2. MCP Client (Consuming External MCP Servers)

_(Also missing vs OpenClaw, but Hermes takes it further)_

Hermes can connect to any MCP server via stdio or HTTP transport and surface those
tools in its registry with per-server filtering. PureClaw can neither expose its
tools via MCP nor consume external MCP servers.

### 3. Subagent Delegation / Multi-Agent Orchestration

Hermes has a `delegate_task` tool that spawns isolated child agent instances with
restricted toolsets and independent terminal sessions (3 concurrent by default).
This is hierarchical task decomposition within the agent itself — the LLM decides
to delegate subtasks. PureClaw has no concept of spawning child agents. Its
harness system can talk to external tools like Claude Code, but the agent can't
spin up copies of itself for parallel work.

### 4. Programmatic Tool Calling (execute_code)

Hermes has an `execute_code` tool that lets the agent write Python scripts which
call Hermes tools programmatically via RPC. This collapses multi-step tool
pipelines into single inference calls — the agent writes a script that orchestrates
tools rather than calling them one at a time. PureClaw has no equivalent; all tool
calls go through the LLM turn-by-turn.

### 5. Image Generation

Hermes integrates FAL.ai with eight image generation models (FLUX 2 variants,
GPT-Image, Ideogram, Recraft, etc.). PureClaw has no image generation capability.

### 6. Voice / Speech I/O

_(Also missing vs OpenClaw)_

Hermes has full voice mode across CLI and messaging platforms: microphone input,
Discord voice channels, and TTS output via five providers (Edge TTS, ElevenLabs,
OpenAI TTS, MiniMax, NeuTTS). PureClaw is text-only.

### 7. Sandboxed Execution / Terminal Backends

_(Also missing vs OpenClaw, but Hermes has a richer model)_

Hermes supports six terminal backends: local, Docker, SSH, Daytona (serverless
persistence), Singularity, and Modal (serverless, hibernates when idle). PureClaw
executes everything on the host with allow-list security only.

### 8. Skill Registry / Community Skills

_(Also missing vs OpenClaw)_

Hermes is compatible with the agentskills.io open standard and can import OpenClaw
skills. Skills self-improve during use and can be autonomously generated by the
agent after solving complex tasks. PureClaw has no skill marketplace or
auto-generation.

### 9. Autonomous Skill Creation / Self-Improvement

Hermes can autonomously create new skills after completing complex tasks and
self-improve existing skills during use. This is a closed learning loop — the agent
gets better at recurring tasks without user intervention. PureClaw has memory
(store/recall) but no procedural memory that captures *how* to do things as
reusable routines.

### 10. IDE Integration (ACP / Editor Protocol)

Hermes exposes itself as an editor-native agent over stdio/JSON-RPC for VS Code,
Zed, and JetBrains. This lets users interact with Hermes from within their editor
with integrated chat, tool activity, diffs, and terminal rendering. PureClaw has
no IDE integration.

### 11. API Server (OpenAI-Compatible Endpoint)

Hermes can expose itself as an OpenAI-compatible HTTP endpoint, allowing integration
with Open WebUI, LobeChat, LibreChat, and any OpenAI-compatible client. PureClaw's
gateway serves pairing and OAuth only — it can't be used as a drop-in LLM endpoint.

### 12. Credential Pools / Provider Failover

Hermes distributes requests across multiple API keys for the same provider with
automatic rotation on rate limits/failures, plus automatic failover to backup
providers. PureClaw has a single API key per provider and no failover mechanism.

### 13. Context References ("@" Notation)

Hermes supports `@file`, `@folder`, `@diff`, `@url` notation in messages that
automatically expands to inject content into context. PureClaw has no equivalent
shorthand — users must use slash commands or tools to bring content into context.

### 14. Checkpoints / Rollback

Hermes automatically snapshots the working directory before file modifications,
enabling `/rollback` to undo changes. PureClaw relies on git for versioning but
has no automatic checkpoint/rollback mechanism.

### 15. RL Training / Trajectory Generation

Hermes has built-in infrastructure for batch trajectory generation, Atropos RL
environments, and trajectory compression for training tool-calling models. This is
research infrastructure for improving the underlying models. PureClaw has nothing
in this space (and this is likely out of scope for PureClaw's goals).

### 16. Webhook / Event-Driven Triggers

_(Also missing vs OpenClaw)_

Hermes has event hooks at lifecycle points (gateway logging, alerts, webhooks) and
plugin-level tool interception. PureClaw has cron for time-based triggers but no
webhook ingestion or event hooks.

### 17. Plugin Architecture

Hermes has a formal plugin system with discovery across three locations, a context
API for registration, and two specialized plugin types (memory providers, context
engines). Plugins can register tools, hooks, and CLI commands. PureClaw has a
`PluginHandle` but it's minimal — there's no plugin discovery, no plugin API, and
no third-party plugin ecosystem.

### 18. Home Assistant / IoT Integration

Hermes has dedicated `ha_*` tools for Home Assistant, enabling smart home control
(lights, switches, sensors, automations). PureClaw has no IoT integration.
